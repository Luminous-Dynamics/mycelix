### 2. CanaryCNN Inference
**File**: `src/canary_cnn.rs`
**Commit**: `mode3-vsv-day2-canarycnn-inference`

**Architecture** (MNIST):
```
Input: 28x28 grayscale
Conv2D(1→16, 3x3) → ReLU → MaxPool(2x2)
Conv2D(16→32, 3x3) → ReLU → MaxPool(2x2)
Flatten → FC(800→128) → ReLU → FC(128→10)
Output: logits [10]
```

**Implementation**:
- `fn conv2d(input: &[Fixed], weights: &[Fixed], ...) -> Vec<Fixed>`
- `fn relu(x: &[Fixed]) -> Vec<Fixed>`
- `fn maxpool2d(x: &[Fixed], ...) -> Vec<Fixed>`
- `fn fc_layer(input: &[Fixed], weights: &[Fixed], bias: &[Fixed]) -> Vec<Fixed>`
- `fn forward(input: &[Fixed; 784]) -> [Fixed; 10]`

**Canonical Loss**: Compute canary loss deltas with MSE-on-logits; postpone any cross-entropy variants until a dedicated cost analysis.

**Testing**:
- Test each layer independently
- Full forward pass on 10 MNIST samples
- Compare with PyTorch output (ε = 0.01)

## 🎓 Reference Materials

### Q16.16 Fixed-Point Format
- Integer part: 16 bits (sign + 15 bits value)
- Fractional part: 16 bits
- Range: -32768.0 to 32767.99998
- Precision: ~0.000015 (2^-16)

### CanaryCNN Purpose (Context)
- Small frozen CNN for gradient quality validation
- Used in VSV protocol to detect Byzantine nodes
- Must be deterministic and verifiable in zkVM
