# Mycelix Emergent Collective Intelligence Guide

## Philosophy: The Wisdom of the Whole

> "The forest knows things no single tree can know. The flock navigates paths no single bird could find. The mycelium network solves problems beyond any individual node. So too with human communities—when we coordinate well, something greater than us emerges."

This guide explores how collective intelligence emerges in Mycelix communities, how to cultivate it, and how to recognize when it's happening.

---

## Architecture Alignment

> **Reference**: See [ARCHITECTURE_ALIGNMENT_GUIDE.md](../ARCHITECTURE_ALIGNMENT_GUIDE.md) for complete alignment standards.

### Epistemic Classification (LEM v2.0)

| Data Type | E-Tier | N-Tier | M-Tier | Description |
|-----------|--------|--------|--------|-------------|
| Collective Insight | E1 (Testimonial) | N1 (Communal) | M2 (Persistent) | Synthesized group wisdom |
| Pattern Detection | E2 (Privately Verifiable) | N1 (Communal) | M2 (Persistent) | Detected patterns in community data |
| Sensing Data | E1-E2 | N0-N1 | M1 (Temporal) | Real-time collective sensing inputs |
| Wisdom Nugget | E3 (Cryptographically Proven) | N2 (Network) | M2 (Persistent) | Validated, crystallized insights |
| CI Assessment | E2 (Privately Verifiable) | N1 (Communal) | M2 (Persistent) | Collective intelligence metrics |

### Token System Integration

- **CIV (Civic Standing)**: Weights contribution to collective sensemaking; higher CIV = greater synthesis influence
- **CGC (Civic Gifting Credit)**: Recognize exceptional insights and facilitation of emergence
- **FLOW**: Optional resource allocation for collective intelligence infrastructure

### Governance Tier Mapping

| Component | Primary DAO Tier | Authority Level |
|-----------|-----------------|-----------------|
| Sensing Layer | Local DAO | Community-level sensing |
| Sensemaking Layer | Local/Sector DAO | Cross-community synthesis |
| Decision Layer | All tiers | Per decision scope |
| Learning Layer | Regional/Global DAO | Network-wide patterns |
| Emergence hApp | Global DAO | Network intelligence oversight |

### Key Institution References

- **Knowledge Council**: Validates Wisdom Nuggets (N2 authority), oversees epistemic quality
- **Audit Guild**: Reviews collective intelligence metrics for manipulation
- **Community Voice Assembly**: Democratic input into CI priorities

## Appendix: Quick Reference

### Conditions for Collective Intelligence

| Condition | What It Means | How to Cultivate |
|-----------|---------------|------------------|
| Diversity | Different perspectives | Inclusive recruitment, rotation |
| Independence | Free thought | Private reflection before public |
| Decentralization | Distributed knowledge | Information sharing norms |
| Aggregation | Integration mechanisms | Good deliberation processes |
| Trust | Psychological safety | Consistent care and follow-through |
| Purpose | Shared direction | Clear, meaningful goals |

### Warning Signs Checklist

- [ ] Are decisions unanimous without discussion?
- [ ] Are the same people always speaking?
- [ ] Are outside perspectives dismissed?
- [ ] Are mistakes repeated?
- [ ] Is information hoarded?
- [ ] Are subgroups siloed?
- [ ] Is there learned helplessness?

### Facilitation for Emergence Principles

1. **Hold space** without controlling content
2. **Follow energy** where aliveness is
3. **Protect diversity** of voices
4. **Harvest wisdom** as it emerges
5. **Trust the process** and the group
