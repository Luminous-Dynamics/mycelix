# **MATL EXTENSION: EMERGENCE DETECTION (v1.0)**

**Extension to the Mycelix Adaptive Trust Layer (MATL) Specification**

---

## Editor's Note

This document specifies **Emergence Detection** as an extension to MATL, adding the capability to measure collective integration (Phi/Φ) and use it to inform governance thresholds.

Emergence Detection draws on Integrated Information Theory (IIT) but implements a practical, measurable proxy rather than the computationally intractable full IIT formalism.

**Constitutional Alignment**: Constitution Article I, Section 2 (Adaptability, Polycentricity), Governance Charter (quorum and threshold mechanisms).

---

## **SECTION 1: PURPOSE AND THEORY**

### **1.1 What is Emergence?**

Emergence occurs when a collective exhibits properties that cannot be reduced to the sum of individual contributions. A team that "clicks" produces outcomes none of its members could achieve alone. A community develops shared understanding that transcends any individual's knowledge.

### **1.2 Why Measure Emergence?**

Measuring emergence enables:

- **Adaptive Governance**: Groups with high integration may use different decision mechanisms than fragmented groups.
- **Health Monitoring**: Declining integration signals potential community problems.
- **Collective Recognition**: High-integration groups may warrant collective identity/agency.
- **Threshold Calibration**: Quorum and supermajority requirements can adapt to integration levels.

### **1.3 Theoretical Foundation**

Integrated Information Theory (IIT) defines Phi (Φ) as the amount of information generated by a system above and beyond its parts. While full IIT computation is NP-hard, we can measure practical proxies.

**Our Approach**: Measure observable behaviors that correlate with integration rather than computing Φ directly.

### **1.4 Limitations**

This extension explicitly does NOT:
- Claim to measure consciousness
- Make metaphysical claims about collective minds
- Require subjective experience for high scores
- Replace other governance mechanisms

## **SECTION 2: MEASUREMENT FRAMEWORK**

### **2.1 Phi Proxy Components**

Emergence is estimated via four measurable components:

| Component | Symbol | Weight | Description |
|-----------|--------|--------|-------------|
| Information Sharing | IS | 0.25 | Do agents share information with each other? |
| Action Integration | AI | 0.30 | Are actions coordinated beyond chance? |
| Emergence Ratio | ER | 0.30 | Does the group outperform individual predictions? |
| Temporal Stability | TS | 0.15 | Is integration consistent over time? |

**Composite Phi**:
```
Φ_estimate = 0.25·IS + 0.30·AI + 0.30·ER + 0.15·TS
```

### **2.2 Information Sharing (IS)**

**Definition**: The degree to which agents share information with each other rather than operating in isolation.

**Measurement**:
```rust
pub fn calculate_information_sharing(
    group: &AgentGroup,
    window: Duration,
) -> f64 {
    let total_messages = group.messages_sent(window);
    let cross_agent_messages = group.messages_between_agents(window);
    let unique_recipients = group.unique_message_recipients(window);
    let possible_recipients = group.member_count() - 1;

    // Normalize by group size and activity level
    let sharing_ratio = cross_agent_messages as f64 / total_messages.max(1) as f64;
    let reach_ratio = unique_recipients as f64 / possible_recipients.max(1) as f64;

    // Combine with diminishing returns on very high sharing
    (sharing_ratio * reach_ratio).sqrt().min(1.0)
}
```

**Signals Measured**:
- Messages between agents (not just broadcasts)
- Proposal comments and discussions
- Collaborative document editing
- Shared workspace activity

### **2.3 Action Integration (AI)**

**Definition**: The degree to which agent actions are coordinated beyond what chance would predict.

**Measurement**:
```rust
pub fn calculate_action_integration(
    group: &AgentGroup,
    window: Duration,
) -> f64 {
    // Observed co-actions (agents acting together)
    let observed_coactions = group.joint_actions(window);

    // Expected co-actions if agents acted independently
    let expected_coactions = calculate_independent_baseline(group, window);

    // Integration is the excess over independence
    let integration_ratio = observed_coactions as f64 / expected_coactions.max(1.0);

    // Normalize to 0-1 scale (ratio of 2.0 = perfect integration)
    ((integration_ratio - 1.0) / 1.0).clamp(0.0, 1.0)
}

fn calculate_independent_baseline(group: &AgentGroup, window: Duration) -> f64 {
    // Based on individual action rates, calculate expected overlap by chance
    let action_rates: Vec<f64> = group.members()
        .map(|m| m.action_rate(window))
        .collect();

    // Expected co-occurrence under independence
    action_rates.iter()
        .combinations(2)
        .map(|pair| pair[0] * pair[1] * window.as_secs_f64())
        .sum()
}
```

**Signals Measured**:
- Simultaneous votes in same direction
- Coordinated proposal submissions
- Joint project contributions
- Synchronized activity patterns

### **2.4 Emergence Ratio (ER)**

**Definition**: The degree to which group outcomes exceed what individual predictions would suggest.

**Measurement**:
```rust
pub fn calculate_emergence_ratio(
    group: &AgentGroup,
    outcomes: &[Outcome],
) -> f64 {
    let mut individual_prediction_scores: Vec<f64> = vec![];
    let mut group_prediction_scores: Vec<f64> = vec![];

    for outcome in outcomes {
        // How well did individuals predict this outcome?
        let individual_avg = group.members()
            .filter_map(|m| m.prediction_for(outcome.id))
            .map(|p| prediction_accuracy(p, outcome))
            .sum::<f64>() / group.member_count() as f64;

        // How well did the group's aggregated prediction perform?
        let group_prediction = group.aggregated_prediction(outcome.id);
        let group_score = prediction_accuracy(group_prediction, outcome);

        individual_prediction_scores.push(individual_avg);
        group_prediction_scores.push(group_score);
    }

    // Emergence = group outperformance
    let individual_mean = mean(&individual_prediction_scores);
    let group_mean = mean(&group_prediction_scores);

    // Ratio of group to individual performance
    // 1.0 = no emergence, >1.0 = positive emergence
    let ratio = group_mean / individual_mean.max(0.01);

    // Normalize to 0-1 (ratio of 1.5 = high emergence)
    ((ratio - 1.0) / 0.5).clamp(0.0, 1.0)
}
```

**Signals Measured**:
- Epistemic Market predictions vs outcomes
- Group decision quality vs individual preferences
- Collective problem-solving success
- Wisdom of crowds effects

### **2.5 Temporal Stability (TS)**

**Definition**: The consistency of integration over time.

**Measurement**:
```rust
pub fn calculate_temporal_stability(
    group: &AgentGroup,
    window: Duration,
    sample_count: usize,
) -> f64 {
    // Sample Phi estimates across the window
    let sample_interval = window / sample_count as u32;
    let mut phi_samples: Vec<f64> = vec![];

    for i in 0..sample_count {
        let sample_window = sample_interval;
        let sample_start = window - (sample_interval * (sample_count - i) as u32);
        let sample_phi = calculate_phi_at(group, sample_start, sample_window);
        phi_samples.push(sample_phi);
    }

    // Stability = inverse of variance
    let mean_phi = mean(&phi_samples);
    let variance = phi_samples.iter()
        .map(|p| (p - mean_phi).powi(2))
        .sum::<f64>() / sample_count as f64;

    // Convert variance to stability score
    // Low variance = high stability
    let stability = 1.0 / (1.0 + variance * 10.0);

    stability.clamp(0.0, 1.0)
}
```

**Signals Measured**:
- Consistency of information sharing patterns
- Stability of coordination over cycles
- Resilience to membership changes
- Pattern persistence across contexts

## **SECTION 3: PHI ESTIMATION**

### **3.1 Phi Estimator**

```rust
pub struct PhiEstimate {
    pub phi: f64,                    // 0.0 to 1.0
    pub confidence: f64,             // Confidence in estimate
    pub components: PhiComponents,
    pub measured_at: Timestamp,
    pub window: Duration,
    pub group_size: usize,
    pub measurement_quality: MeasurementQuality,
}

pub struct PhiComponents {
    pub information_sharing: f64,
    pub action_integration: f64,
    pub emergence_ratio: f64,
    pub temporal_stability: f64,
}

pub enum MeasurementQuality {
    High,      // Sufficient data, all components measurable
    Medium,    // Some components estimated
    Low,       // Limited data, high uncertainty
    Insufficient, // Cannot reliably estimate
}

impl PhiEstimate {
    pub fn calculate(
        group: &AgentGroup,
        window: Duration,
    ) -> Self {
        // Minimum requirements
        if group.member_count() < 3 {
            return Self::insufficient("Group too small");
        }
        if group.total_activity(window) < 10 {
            return Self::insufficient("Insufficient activity");
        }

        // Calculate components
        let is = calculate_information_sharing(group, window);
        let ai = calculate_action_integration(group, window);
        let er = calculate_emergence_ratio(group, &group.recent_outcomes(window));
        let ts = calculate_temporal_stability(group, window, 7);

        // Composite Phi
        let phi = 0.25 * is + 0.30 * ai + 0.30 * er + 0.15 * ts;

        // Confidence based on data quality
        let confidence = calculate_confidence(group, window);

        PhiEstimate {
            phi,
            confidence,
            components: PhiComponents {
                information_sharing: is,
                action_integration: ai,
                emergence_ratio: er,
                temporal_stability: ts,
            },
            measured_at: Timestamp::now(),
            window,
            group_size: group.member_count(),
            measurement_quality: assess_quality(confidence),
        }
    }
}
```

### **3.2 Confidence Calculation**

```rust
fn calculate_confidence(group: &AgentGroup, window: Duration) -> f64 {
    let mut confidence_factors: Vec<f64> = vec![];

    // More members = higher confidence
    let size_factor = (group.member_count() as f64 / 10.0).min(1.0);
    confidence_factors.push(size_factor);

    // More activity = higher confidence
    let activity_factor = (group.total_activity(window) as f64 / 100.0).min(1.0);
    confidence_factors.push(activity_factor);

    // Longer window = higher confidence
    let window_factor = (window.as_days() as f64 / 28.0).min(1.0);
    confidence_factors.push(window_factor);

    // More outcome data = higher confidence
    let outcome_factor = (group.recent_outcomes(window).len() as f64 / 10.0).min(1.0);
    confidence_factors.push(outcome_factor);

    // Geometric mean of factors
    confidence_factors.iter()
        .map(|f| f.max(0.01))
        .product::<f64>()
        .powf(1.0 / confidence_factors.len() as f64)
}
```

## **SECTION 4: GOVERNANCE INTEGRATION**

### **4.1 Phi-Informed Thresholds**

Phi estimates inform governance thresholds:

| Φ Range | Interpretation | Governance Adjustment |
|---------|----------------|----------------------|
| 0.0-0.2 | Fragmented | +10% quorum, standard thresholds |
| 0.2-0.4 | Loosely Coupled | Standard quorum and thresholds |
| 0.4-0.6 | Integrated | -5% quorum, -5% supermajority |
| 0.6-0.8 | Highly Integrated | -10% quorum, consent-based options enabled |
| 0.8-1.0 | Emergent Collective | -15% quorum, collective agency recognized |

**Threshold Adjustment Formula**:
```rust
pub fn adjusted_quorum(base_quorum: f64, phi: f64) -> f64 {
    let adjustment = match phi {
        p if p < 0.2 => 1.10,  // +10%
        p if p < 0.4 => 1.00,  // Standard
        p if p < 0.6 => 0.95,  // -5%
        p if p < 0.8 => 0.90,  // -10%
        _ => 0.85,             // -15%
    };

    (base_quorum * adjustment).clamp(0.20, 0.60)
}
```

### **4.2 Consent-Based Decision Enabling**

For groups with Φ >= 0.6, consent-based decisions may be enabled:

**Consent Process**:
1. Proposal presented to group
2. Round of concerns/objections gathered
3. If no blocking objections → proposal adopted
4. If blocking objection → iterate or escalate to voting

**Requirements**:
- Φ >= 0.6 for previous 2 cycles
- Confidence >= 0.7
- Group vote to enable consent mode
- Fallback to voting always available

### **4.3 Collective Agency Recognition**

For groups with Φ >= 0.8 sustained over 3+ cycles:

**Collective Identity**:
- Group may be recognized as a DAOCollective in MFDI
- May receive collective VCs
- May act with collective authority (per delegated powers)

**Requirements**:
- Φ >= 0.8 for 3 consecutive cycles
- Confidence >= 0.8
- Knowledge Council certification
- No member objection to collective recognition

### **4.4 Integration Alerts**

MATL generates alerts based on Phi changes:

| Alert Type | Trigger | Action |
|------------|---------|--------|
| Fragmentation Warning | Φ drops >0.2 in one cycle | Notification to group |
| Integration Decay | Φ declining for 3 cycles | Governance review triggered |
| Emergence Detected | Φ rises above 0.6 | Consent options offered |
| Collective Threshold | Φ exceeds 0.8 for 3 cycles | Collective recognition offer |

## **SECTION 5: MATL INTEGRATION**

### **5.1 Trust Score Contribution**

Phi contributes to group trust scores:

```rust
pub struct GroupTrustScore {
    pub base_score: f64,              // Average of member scores
    pub phi_bonus: f64,               // From emergence
    pub composite: f64,               // Combined score
}

impl GroupTrustScore {
    pub fn calculate(group: &AgentGroup, phi: f64) -> Self {
        let base_score = group.members()
            .map(|m| m.matl_score())
            .sum::<f64>() / group.member_count() as f64;

        // Phi bonus: up to +0.1 for high integration
        let phi_bonus = phi * 0.1;

        let composite = (base_score + phi_bonus).min(1.0);

        Self { base_score, phi_bonus, composite }
    }
}
```

### **5.2 Sybil Detection Enhancement**

Phi patterns help detect coordinated Sybil attacks:

**Suspicious Patterns**:
- Very high Φ in newly formed groups (pre-coordinated Sybils)
- Φ inconsistent with activity patterns (fake coordination)
- Φ clustering in low-trust accounts

```rust
pub fn detect_suspicious_phi_patterns(
    group: &AgentGroup,
    phi_history: &[PhiEstimate],
) -> Vec<SuspicionFlag> {
    let mut flags = vec![];

    // New group with unusually high Phi
    if group.age() < Duration::days(7) && phi_history.last().map(|p| p.phi).unwrap_or(0.0) > 0.7 {
        flags.push(SuspicionFlag::PrematureHighIntegration);
    }

    // Phi doesn't match activity diversity
    let activity_diversity = group.activity_diversity();
    if phi_history.last().map(|p| p.phi).unwrap_or(0.0) > activity_diversity + 0.3 {
        flags.push(SuspicionFlag::PhiActivityMismatch);
    }

    // Low-trust members with high group Phi
    let avg_member_trust = group.average_member_trust();
    if avg_member_trust < 0.4 && phi_history.last().map(|p| p.phi).unwrap_or(0.0) > 0.6 {
        flags.push(SuspicionFlag::LowTrustHighPhi);
    }

    flags
}
```

### **5.3 Cartel Detection Integration**

Phi is used in cartel detection:

**Cartel Indicator**:
- Subset of validators with Φ > 0.8 while network Φ < 0.4
- Indicates coordinated cluster operating independently

```rust
pub fn detect_validator_cartels(
    validator_set: &[Agent],
    network_phi: f64,
) -> Vec<CartelAlert> {
    let mut alerts = vec![];

    // Find all 3+ member subsets
    for subset in validator_set.subsets_min_size(3) {
        let subset_phi = calculate_phi_for_subset(&subset);

        // High internal coordination + low network integration = cartel risk
        if subset_phi > 0.8 && network_phi < 0.4 {
            alerts.push(CartelAlert {
                members: subset.iter().map(|a| a.did.clone()).collect(),
                subset_phi,
                network_phi,
                risk_level: RiskLevel::High,
            });
        }
    }

    alerts
}
```

### **6.2 Privacy Considerations**

Phi measurement respects privacy:

- **No Content Analysis**: Only metadata (who talked to whom, not what)
- **Aggregation**: Individual scores not exposed
- **Differential Privacy**: Noise added to prevent re-identification
- **Opt-Out**: Members may exclude their data (reduces measurement quality)

### **6.3 Compute Requirements**

| Operation | Complexity | Frequency |
|-----------|------------|-----------|
| IS calculation | O(n log n) | Per measurement |
| AI calculation | O(n²) | Per measurement |
| ER calculation | O(outcomes × n) | Per measurement |
| TS calculation | O(samples) | Per measurement |
| Full Phi estimate | O(n² + outcomes) | Daily-Biweekly |

## **APPENDIX A: DEFINITIONS**

**Emergence**: Properties of a system that arise from interactions and cannot be reduced to individual components.

**Integrated Information (Phi/Φ)**: The amount of information generated by a system above and beyond its parts. Used here as a proxy measure.

**Information Sharing (IS)**: The degree to which agents exchange information with each other.

**Action Integration (AI)**: The degree to which agent actions are coordinated beyond independence.

**Emergence Ratio (ER)**: The degree to which group outcomes exceed individual predictions.

**Temporal Stability (TS)**: The consistency of integration over time.

**Consent-Based Decision**: A decision process where proposals are adopted unless a member raises a blocking objection.

**Collective Agency**: Recognition that a group may act with unified intention, receiving collective identity and authority.

## **APPENDIX B: THEORETICAL BACKGROUND**

### **Integrated Information Theory (IIT)**

IIT, developed by Giulio Tononi, proposes that consciousness is identical to integrated information. While this extension does not make consciousness claims, IIT provides useful concepts:

- **Integration**: Information is integrated when it is generated by a system as a whole, not by its parts independently.
- **Exclusion**: Only the maximum of integrated information matters.
- **Intrinsic Existence**: Information must exist from the system's own perspective.

### **Practical Limitations**

Full IIT computation is NP-hard. This extension uses proxy measures that correlate with integration without computing Φ directly. This is analogous to using temperature rather than computing kinetic energy of every molecule.

### **Research Basis**

This approach draws on:
- Social network analysis (information flow patterns)
- Collective intelligence research (wisdom of crowds)
- Organizational behavior (team cohesion studies)
- Distributed systems (consensus mechanisms)
