# Epistemic Charter

The foundational principles governing knowledge in the Mycelix ecosystem.

## Preamble

We, the participants in the Mycelix Knowledge network, establish this charter to govern the creation, verification, and use of knowledge in our decentralized commons. We recognize that knowledge is a public good, that truth matters, and that epistemological humility is essential.

## Core Principles

### 1. Truth-Seeking
**We commit to seeking truth through rigorous verification.**

- Claims should be falsifiable where possible
- Evidence should be provided and checkable
- Epistemic levels reflect actual verifiability, not aspiration
- Contradictions should be resolved, not ignored

### 2. Transparency
**We commit to transparency in knowledge claims.**

- Sources must be cited
- Methodology must be explained
- Limitations must be acknowledged
- Conflicts of interest must be disclosed

### 3. Humility
**We acknowledge the limits of knowledge.**

- High epistemic levels (E > 0.8) require extraordinary evidence
- Uncertainty is encoded, not hidden
- Claims can be updated as evidence changes
- "I don't know" is a valid position

### 4. Accessibility
**Knowledge should be accessible to all.**

- Clear language over jargon
- Multiple verification paths
- Progressive complexity
- Inclusive participation

### 5. Decentralization
**No single authority controls truth.**

- Distributed verification through markets
- MATL-weighted consensus
- Byzantine fault tolerance
- Censorship resistance

## Epistemic Classification Standards

### Empirical (E) Axis Requirements

| Level | Requirement | Evidence Needed |
|-------|-------------|-----------------|
| E0 (0.0-0.2) | Self-attestation | None required |
| E1 (0.2-0.4) | Testimonial | 2+ independent witnesses |
| E2 (0.4-0.6) | Verifiable credentials | Cryptographic proof |
| E3 (0.6-0.8) | On-chain verification | Blockchain evidence |
| E4 (0.8-1.0) | Scientific measurement | Reproducible methodology |

### Normative (N) Axis Standards

| Level | Scope | Consensus Mechanism |
|-------|-------|---------------------|
| N0 (0.0-0.25) | Personal | Self-declaration |
| N1 (0.25-0.5) | Community | Community vote |
| N2 (0.5-0.75) | Network | MATL-weighted oracle |
| N3 (0.75-1.0) | Universal | Formal verification |

### Mythic (M) Axis Persistence

| Level | Persistence | Example |
|-------|-------------|---------|
| M0 (0.0-0.25) | Ephemeral | Meeting notes |
| M1 (0.25-0.5) | Archived | Quarterly reports |
| M2 (0.5-0.75) | Permanent | Historical records |
| M3 (0.75-1.0) | Foundational | Constitutional docs |

## Verification Standards

### Market-Based Verification
- Verification markets require minimum stake
- Oracle participation requires minimum MATL
- Resolution uses quadratic MATL weighting
- 45% Byzantine fault tolerance

### Evidence Standards
- Primary sources preferred over secondary
- Peer review increases credibility
- Cross-references strengthen claims
- Cryptographic proofs are strongest

## Rights and Responsibilities

### Claim Authors
**Rights:**
- Submit claims to the network
- Update claims with new evidence
- Request verification markets
- Dispute resolutions

**Responsibilities:**
- Accurate epistemic classification
- Provide supporting evidence
- Respond to contradictions
- Update retracted claims

### Verifiers (Oracles)
**Rights:**
- Participate in verification markets
- Stake reputation on votes
- Earn rewards for accuracy
- Dispute resolution processes

**Responsibilities:**
- Honest assessment
- Evidence-based voting
- Transparency in reasoning
- Recusal from conflicts

### Users
**Rights:**
- Query the knowledge graph
- Request fact-checks
- Challenge low-credibility claims
- Access verification history

**Responsibilities:**
- Not misrepresent verdicts
- Report errors found
- Respect intellectual property
- Follow usage guidelines

## Governance

### Amendment Process
1. Proposal submitted with rationale
2. 30-day community discussion
3. MATL-weighted vote
4. 2/3 majority required
5. Implementation period

### Dispute Resolution
1. Initial mediation
2. Oracle review panel
3. Governance escalation
4. Community arbitration

## Enforcement

### Claim Violations
- Fraudulent claims: Author reputation penalty
- Repeated false claims: Temporary suspension
- Malicious manipulation: Permanent removal

### Oracle Violations
- Consistent inaccuracy: MATL reduction
- Collusion: Stake forfeiture
- Manipulation: Network exclusion

## Sunset Clause

This charter shall be reviewed every two years. Major changes require supermajority approval. The charter remains in effect until formally superseded.
