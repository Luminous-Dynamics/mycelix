## Introduction

Onboarding in Epistemic Markets is not just about learning features—it's about joining a community of truth-seekers. We design onboarding to:

1. **Welcome**: Make everyone feel they belong
2. **Educate**: Teach core concepts gradually
3. **Activate**: Get to that first meaningful prediction
4. **Support**: Provide help when needed
5. **Connect**: Build community bonds early

This document details the onboarding experience for each persona and the systems that support their journey.

---

## Part I: Onboarding Philosophy

### Principles

**1. Progressive Disclosure**
- Show only what's needed now
- Reveal complexity as mastery grows
- Never overwhelm

**2. Learn by Doing**
- Theory follows practice
- First prediction before deep explanation
- Safe space for mistakes

**3. Community First**
- Connect with humans, not just software
- Mentorship available from day one
- No one learns alone

**4. Multiple Paths**
- Different personas need different journeys
- Self-paced with support available
- Skip what you know, dive deep on what interests you

**5. Celebration of Progress**
- Acknowledge milestones
- Make learning visible
- Reward engagement

### Anti-Patterns to Avoid

| Avoid | Instead |
|-------|---------|
| Wall of text on signup | Brief welcome, learn by doing |
| Mandatory tutorials before access | Optional tutorials, guided first actions |
| Same path for everyone | Adaptive paths based on experience |
| Dumping all features at once | Progressive feature revelation |
| Sink or swim | Mentorship and support available |
| Gamification addiction | Meaningful achievements |

```
How would you like to participate?

○ Explorer
  "I want to browse and learn before participating"

○ Predictor
  "I'm ready to start making predictions"

○ Expert
  "I have domain expertise to contribute"

○ Builder
  "I want to create markets for my community"
```

### Step 3: Brief Orientation (2 minutes)

Based on selection, show 3-5 key concepts:

**For Explorers**:
1. What is a prediction market?
2. How to browse markets
3. What reputation means
4. Where to learn more

**For Predictors**:
1. How predictions work
2. What you can stake
3. How resolution works
4. Your first prediction

**For Experts**:
1. Oracle opportunities
2. Reputation domains
3. Creating evidence
4. Your expertise profile

**For Builders**:
1. Market creation basics
2. Community integration
3. Question markets
4. Your first market

### Step 4: First Action (3-5 minutes)

Get to meaningful action quickly:

**Explorers**: Browse 3 interesting markets, maybe signal curiosity
**Predictors**: Make first prediction (on a simple, low-stakes market)
**Experts**: Set up expertise profile, view relevant markets
**Builders**: Preview market creation, maybe create a question

### Step 5: Connection (Optional, 2 minutes)

```
You're off to a great start!

Would you like to:

○ Connect with a mentor
  Get personal guidance from an experienced member

○ Join the newcomers community
  Connect with others just starting out

○ Subscribe to learning emails
  Weekly tips and guidance

○ Continue on my own
  You can always access these later
```

All onboarding experiences must be accessible:

**Screen Reader Support**:
- All tutorials have text alternatives
- Proper heading structure

- ARIA labels throughout

**Keyboard Navigation**:
- Complete keyboard accessibility
- Focus management
- Shortcut keys documented
- Skip links available

**Visual Accessibility**:
- High contrast options
- Scalable text
- No color-only information
- Clear visual hierarchy

**Cognitive Accessibility**:
- Plain language
- Clear steps
- Progress indicators
- Break options

**Motor Accessibility**:
- No time limits
- Large click targets
- Pause/resume capability
- Alternative input methods

## Part X: Onboarding Content Guidelines

### Voice and Tone

**Welcome**: Warm, inviting, personal
**Learning**: Clear, patient, encouraging
**Celebration**: Genuine, not over-the-top
**Guidance**: Helpful, not condescending
**Challenge**: Inspiring, not intimidating

### Writing Principles

1. **Brevity**: Say less, mean more
2. **Clarity**: No jargon without explanation
3. **Action**: Focus on what to do
4. **Respect**: Never talk down
5. **Honesty**: Don't overpromise

### Examples

**Good**:
"Make your first prediction. It's okay if you're wrong—that's how we learn."

**Bad**:
"Welcome to your epistemic journey! In this revolutionary platform, you'll engage in probabilistic reasoning to collectively determine truth through market mechanisms."
